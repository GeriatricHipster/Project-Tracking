# Render deployment checklist

Use this checklist after the project files are uploaded to GitHub.

## Required repository layout

The GitHub repository top level must contain:

```text
client
server
package.json
render.yaml
README.md
```

If `render.yaml` is inside a folder, the Blueprint deployment will not work.

## Render Blueprint should create

```text
buildtrack-cloud   web service
buildtrack-db      PostgreSQL database
```

## Build settings are already in render.yaml

You should not need to manually enter build commands in Render when using Blueprint deployment.

The web service uses:

```text
Build Command: npm run install:all && npm run build
Start Command: npm start
Health Check Path: /api/health
```

## Environment variables are already in render.yaml

The Blueprint sets:

```text
DATABASE_URL      created from the Render PostgreSQL database
JWT_SECRET        generated by Render
CLIENT_ORIGIN     *
NODE_ENV          production
PGSSLMODE         require
APP_NAME          BuildTrack Cloud
EMAIL_PROVIDER    none
```

## Optional invitation email settings

After the app is live, you can add Resend or SMTP email settings in Render so project invitations are emailed when users are assigned.

Open:

```text
EMAIL_INVITATION_SETUP.md
RESEND_EMAIL_SETUP.md
```

The app works without those settings. Project members can still be assigned, but invitation emails will not send until email is configured.

## First live test

After deployment, open:

```text
https://your-render-service-name.onrender.com/api/health
```

Expected result:

```json
{"ok":true,"service":"buildtrack-server"}
```

Then open the main app URL:

```text
https://your-render-service-name.onrender.com
```

Register your first real user account.
